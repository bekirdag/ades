## Relationship Family Source-Lane Inventory

Work item NSC-A001 requires the inventory to be organized by relationship
family, not only by current builder. Several families are implemented today as
subfamilies inside shared ADES builders; those rows are marked as embedded and
should not be interpreted as separate CLI lanes until NSC-A002/NSC-A003 split
schemas further.

Common market-graph lane behavior:

- Market edge lanes emit `impact_nodes.tsv`, `impact_edges.tsv`, and
  `manifest.json`; when `--build-artifact` is supplied they also write
  `market_graph_store.sqlite` plus `market_graph_store_manifest.json`.
  Default source-lane outputs are under
  `/mnt/githubActions/ades_big_data/pack_sources/impact_relationships/<lane>/<run_id>`;
  built market graph artifacts are under
  `/mnt/githubActions/ades_big_data/artifacts/market-graph/<run_id>`.
- The workspace-level storage and manifest contract is recorded in
  [`big_data_workspace_manifest_2026-06-30.json`](big_data_workspace_manifest_2026-06-30.json).
  Lane rows below name the local manifest emitted by each build/fetch command.
- `ades registry validate-market-graph-source-lanes --edge-tsv-path ...`
  checks source attribution and relation metadata for normalized edge TSVs.
- Relation schema version is the current ADES relationship metadata surface in
  `src/ades/impact/relationship_schema.py`; lane and graph manifests use
  `schema_version: 1` and relation validation emits relation warning counts.
- Node type schema version is the current ADES lane manifest surface
  (`schema_version: 1`, with `supported_node_types` where the shared
  program/org builder writes it). Formal cross-lane node type standardization is
  tracked by follow-up NSC-A002.
- Source tier is inferred by `src/ades/impact/source_catalog.py` from source
  name and URL; current TSV rows do not persist an explicit `source_tier`
  column.
- Required evidence currently lives in normalized columns such as `source_name`,
  `source_url`, `source_snapshot`, `source_year`, `source_kind`, `notes`,
  `confidence`, `event_types`, and `direction_preconditions`. Stronger
  mandatory evidence enforcement, explicit promotion rules, and conflict
  blocking are follow-up NSC-A004 through NSC-A009 work.

## Source-Tier Policy Summary

Market impact edge lanes share one policy surface:

- `src/ades/impact/source_catalog.py` classifies source tiers from `source_name` and `source_url`.
- Promotion-friendly tiers include official/high-trust source families such as government, regulator, exchange, issuer-disclosed, licensed, industry-association, reviewed-proposal, and local pack metadata.
- `validate_source_attribution()` warns on missing source fields, unknown source tiers, Wikidata-only bridge sources, and test fixtures.
- `src/ades/impact/source_lane_validation.py` reports `source_tier_counts`, `source_warning_counts`, and `relation_warning_counts` for edge TSV lanes.
- `src/ades/impact/graph_builder.py` carries the same counts and warnings into `market_graph_store_manifest.json`.

Important gap for follow-up work: current market graph edge TSVs do not carry an
explicit `source_tier` column. Tier is inferred at validation/build time from
source name and URL, so future production promotion rules should either persist
the inferred tier or add an explicit reviewed tier field before release gating.
